Características

Diseñado para trazabilidad total

Cada decisión, cada línea de código, cada conversación — todo queda registrado y trazable.

①

Spec Driven Development

Flujo completo: discovery → spec → PRD → tasks → architecture → implementation → testing → review → commit → push → documentation → memory update.

②

Trazabilidad total

Cada task tiene un ID único. Los commits usan Conventional Commits con task-id. TODO conectado al PRD original.

③

8 Agentes OpenCode

Orquestador + implementation + frontend + DB + testing + GitHub + review + documentation. Cada uno con permisos y contexto limitado.

④

PostgreSQL fuente de verdad

Todo se persiste en PostgreSQL 16 con SQLAlchemy. Migraciones con Alembic. La base de datos es el único fuente de verdad.

⑤

Conventional Commits

Commits con formato tipo(scope): descripción [task-XXX]. Trazabilidad automática entre código y requerimientos.

⑥

Architecture Decision Records

Cada decisión técnica se registra con contexto, opciones consideradas y justificación. Historial completo del porqué.

⑦

State Machine Resumible

Cada paso del flujo se persiste en flow_steps. Si el contexto del LLM se pierde, se retoma desde donde quedó.

⑧

Vault Bidireccional

Sincroniza la DB con Obsidian: exporta a markdown con wikilinks y restaura la DB desde las notas. Sin pérdida de datos.

⑨

Enums + CheckConstraints

8 enums tipados con restricciones a nivel DB para mantener integridad de datos. Sin strings mágicos.

⑩

41 Tests + CI Automático

Suite de 41 tests con pytest en SQLite in-memory + CI en GitHub Actions con matrix Python 3.11/3.12/3.13.

⑪

7 Skills Modulares

Skills especializadas por área: frontend, backend, database, testing, code-review, git-workflow, documentation.

⑫

Context7 MCP

Documentación en tiempo real de librerías y frameworks vía MCP server. Consulta docs sin salir del editor.

Agentes

Los 8 agentes de Lemoria

Cada agente tiene un rol específico, permisos ajustados y recibe solo el contexto que necesita.

orchestrator

Orquestador

Orquestador

Analiza el contexto, aplica SDD, delega tareas a subagentes y consolida resultados. Es el director de orquesta.

implementation-agent

Implementation

Implementación

Implementa lógica de backend, scripts, APIs y toda la lógica de negocio. Especializado en traducción de specs a código.

frontend-agent

Frontend

UI / UX

Implementa interfaces de usuario, componentes, estilos y routing. Separa la capa de presentación del backend.

db-agent

Database

Base de datos

Diseña esquemas, genera migraciones Alembic, optimiza consultas y asegura la integridad referencial.

testing-agent

Testing

Pruebas

Escribe tests unitarios y de integración con pytest. Verifica cobertura y que todo pase antes del merge.

github-agent

GitHub

Trazabilidad

Mantiene la trazabilidad con GitHub: issues, PRs, milestones. Solo lectura (bash allow, edit deny).

review-agent

Review

Revisión técnica

Revisa el código, arquitectura y decisiones técnicas. Sin permisos de escritura — solo observa y reporta.

documentation-agent

Documentación

Documentación

Genera y mantiene la documentación en Obsidian vault, READMEs y archivos markdown. Bash + edit allow.

SDD Flow

Spec Driven Development

15 pasos desde la idea hasta la memoria actualizada. Cada paso queda registrado en PostgreSQL.

1 Discovery Preguntas esenciales
2 Spec Especificación detallada
3 PRD Product Requirements
4 Tasks Desglose en tareas
5 Architecture Diseño arquitectónico
6 Implementation Código por agentes
7 Testing Tests unitarios + integración
8 Review Revisión técnica
9 Commit Conventional Commit
10 Push Subida a GitHub
11 Documentation Documentación generada
12 Memory Update Actualizar memoria
13 State Machine flow_steps en DB
Cómo empezar

Listo en 3 pasos

Instala Lemoria, inicializa el proyecto y empieza a orquestar tu desarrollo con IA.

  • 1

    Clona el repositorio

    git clone https://github.com/devcristianlopez/lemoria.git

  • 2

    Instala el paquete

    cd lemoria && pip install -e .

  • 3

    Inicializa tu proyecto

    lemoria init — configura la base de datos, los agentes y empieza a delegar.

$ git clone https://github.com/devcristianlopez/lemoria.git $ cd lemoria $ pip install -e . $ lemoria init $ lemoria project create "Mi proyecto" -d "Descripción" $ lemoria flow start <project-id> "Mi idea"
