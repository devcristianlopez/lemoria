Características

Cada decisión, cada línea de código, cada conversación — todo queda registrado y trazable.

[01]

Spec Driven Development

Flujo completo: discovery → spec → PRD → tasks → architecture → implementation → testing → review → commit → push → documentation → memory.

[02]

Trazabilidad total

Cada task tiene un ID único. Commits con task-XXX. Todo conectado al PRD original.

[03]

8 Agentes OpenCode

Orquestador + implementation + frontend + DB + testing + GitHub + review + documentation. Cada uno con contexto limitado.

[04]

PostgreSQL fuente de verdad

Todo se persiste en PostgreSQL 16 con SQLAlchemy. La base de datos es el único fuente de verdad.

[05]

Conventional Commits

Commits con formato tipo(scope): descripción [task-XXX]. Trazabilidad automática código ↔ requerimientos.

[06]

Architecture Decision Records

Cada decisión técnica se registra con contexto, opciones consideradas y justificación.

[07]

State Machine Resumible

Cada paso del flujo se persiste en flow_steps. Si el contexto del LLM se pierde, se retoma desde donde quedó.

[08]

Vault Bidireccional

Exporta DB a markdown con wikilinks y restaura DB desde notas. Sin pérdida de datos.

[09]

Enums + CheckConstraints

8 enums tipados con restricciones a nivel DB. Sin strings mágicos.

[10]

41 Tests + CI Automático

Suite con pytest en SQLite in-memory + CI en GitHub Actions con matrix Python 3.11/3.12/3.13.

[11]

7 Skills Modulares

Skills especializadas: frontend, backend, database, testing, code-review, git-workflow, documentation.

[12]

Context7 MCP

Documentación en tiempo real de librerías y frameworks vía MCP server.

$ lemoria agents --list

Los 8 agentes

Cada agente tiene un rol específico, permisos ajustados y recibe solo el contexto que necesita.

orchestrator

Orquestador

Orquestador

Analiza el contexto, aplica SDD, delega tareas a subagentes y consolida resultados.

implementation-agent

Implementation

Implementación

Implementa lógica de backend, scripts, APIs y toda la lógica de negocio.

frontend-agent

Frontend

UI / UX

Implementa interfaces de usuario, componentes, estilos y routing.

db-agent

Database

Base de datos

Diseña esquemas, genera migraciones, optimiza consultas y asegura integridad referencial.

testing-agent

Testing

Pruebas

Escribe tests unitarios y de integración con pytest. Verifica cobertura.

github-agent

GitHub

Trazabilidad

Mantiene trazabilidad con GitHub: issues, PRs, milestones. Solo lectura.

review-agent

Review

Revisión técnica

Revisa código, arquitectura y decisiones técnicas. Solo observa y reporta.

documentation-agent

Documentación

Documentación

Genera y mantiene documentación en Obsidian vault, READMEs y markdown.

Listo en 3 pasos

Instala Lemoria, inicializa el proyecto y empieza a orquestar tu desarrollo con IA.

  • step/1

    Clona el repositorio

    git clone https://github.com/devcristianlopez/lemoria.git

  • step/2

    Instala el paquete

    cd lemoria && pip install -e .

  • step/3

    Inicializa tu proyecto

    lemoria init — configura la base de datos, los agentes y empieza a delegar.
