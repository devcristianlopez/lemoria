# ADR: Infraestructura y contenerización

**Status**: proposed
**ID**: 3642855f
**Proyecto**: [[projects/guardian-live-monitor/README|guardian-live-monitor]]

## Description

Se usó Docker Compose con 5 servicios (postgres, redis, backend, detector, dashboard) en una red bridge compartida guardian-net. Healthchecks en postgres y redis para garantizar orden de arranque. Volumen persistente pgdata para PostgreSQL. Dispositivo /dev/video0 mapeado al contenedor detector con restart unless-stopped para resiliencia. El dashboard usa nginx:alpine por ser liviano y eficiente para servir estáticos.

## Rationale

Alternativas: Kubernetes (overkill para MVP), Podman (menos compatible), despliegue manual (inconsistente). Docker Compose es el estándar para MVP mono-host.
