# ADR: MJPEG stream vs browser getUserMedia for live video

**Status**: proposed
**ID**: 5d571a0b
**Proyecto**: [[projects/guardian-live-monitor/README|guardian-live-monitor]]

## Description

Decidido usar MJPEG stream desde el detector (OpenCV) en vez de getUserMedia del navegador. Motivo: el detector y el navegador competían por la misma cámara web (/dev/video0). getUserMedia era poco confiable (timeouts, sin prompt de permiso, sin errores visibles). MJPEG resuelve: 1) un solo cliente de cámara (el detector), 2) stream nativo en navegadores vía <img>, 3) el dashboard muestra exactamente lo que el detector ve.

## Rationale

Alternativas consideradas: getUserMedia (descartado por conflictos de dispositivo y falta de confiabilidad), WebRTC (sobreingeniería para MVP)
