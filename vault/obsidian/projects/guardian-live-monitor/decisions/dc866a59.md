# ADR: Live video stream en dashboard

**Status**: proposed
**ID**: dc866a59
**Proyecto**: [[projects/guardian-live-monitor/README|guardian-live-monitor]]

## Description

Se implementó servidor MJPEG embebido en el detector usando http.server en un thread daemon. El dashboard muestra el stream via <img src=/stream> con proxy inverso de nginx al detector. Cuando llega una alerta via WebSocket, el overlay del video flashea con el color de la severidad. Calidad JPEG 65% para balance entre ancho de banda y claridad. Frame rate limitado a ~20fps por el sleep de 50ms en el handler MJPEG.

## Rationale

Alternativas: WebSocket con frames base64 (alto overhead), HLS/RTSP (complejidad excesiva para MVP), enviar frames por el mismo WebSocket de eventos (mezcla responsabilidades). MJPEG es el estándar más simple y compatible para streams de video en navegadores sin plugins.
