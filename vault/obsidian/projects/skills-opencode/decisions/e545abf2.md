# ADR: Estructura y formato de skills opencode

**Status**: proposed
**ID**: e545abf2
**Proyecto**: [[projects/skills-opencode/README|skills-opencode]]

## Description

Se adopta el formato estándar de opencode: cada skill en su propia carpeta con un archivo SKILL.md que contiene YAML frontmatter (name, description) y cuerpo en markdown. Misma estructura que lemoria y omarchy.

## Rationale

Alternativas consideradas: (1) Un solo archivo .md por skill en la raíz del repo (descartado por inconsistencia con skills existentes). (2) Archivos .yaml separados para metadatos (descartado por sobreingeniería). (3) SKILL.md con frontmatter YAML integrado (seleccionado por consistencia con el ecosistema opencode).
